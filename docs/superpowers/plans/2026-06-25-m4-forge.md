**Files:** `src-tauri/src/ir.rs`, `src-tauri/src/util.rs`

**Acceptance:** compiles; `Artifact` round-trips through serde_json.
**Tests:** `ir.rs` serde round-trip; `util::backup_dir` returns sibling dir; `sha256_hex("")` known value.

### B2 — `artifacts` table columns + Store CRUD
**File:** `src-tauri/src/store.rs`
- In `migrate()`, add idempotent `pragma_table_info('artifacts')` guards to ALTER-ADD `block TEXT` and `pre_image_sha256 TEXT` (mirror the `parent_session_id` pattern). Base 8 columns already exist.
- Add methods: `save_artifact`, `artifact_by_id` (+`.optional()`), `artifacts_for_finding`, `all_artifacts`, `update_artifact_status`.
- IDs via `stable_id([finding_id, kind, target_path, nanos])`.
**Acceptance:** save → get → list → update_status round-trips; new columns present after migrate on a fresh AND a pre-existing db.
**Tests:** in-memory/temp `Store`: round-trip an `Artifact`; status transition pending→applied→reverted persists `applied_at`/`backup_path`/`pre_image_sha256`; `artifacts_for_finding` filters correctly.

### B3 — `forge::apply_block` + `forge::revert_block` (pure write/restore core)
**File:** `src-tauri/src/forge.rs`
- `pub fn apply_block(target: &Path, block: &str) -> Result<ApplyOutcome>` where `ApplyOutcome { changed: bool, backup_path: Option<PathBuf>, pre_image_sha256: String, proposed: String }`. Logic: read current (missing→""); `proposed = ensure_block(current, block)`; if unchanged → `changed:false`, no backup; else write current bytes to `backup_dir/<id>.bak` (caller passes id or fn takes a backup path), record sha, then **atomic write** (temp-in-dir + rename) of `proposed`; `ensure_parent` first.
- `pub fn revert_block(target: &Path, backup_path: &Path, expected_sha: &str) -> Result<()>`: read backup; recompute sha; if `!= expected_sha` → `anyhow!("backup integrity check failed")`; else atomic-write backup bytes over target.
- Keep `ensure_block` private and reused (no duplication).
**Acceptance:** apply is idempotent (2nd call changed=false, single block, one backup); revert restores exact pre-image; sha mismatch refuses and leaves target untouched.
**Tests (all temp-path / injected target — no real config):**
- `apply_block_writes_once_then_noop`
- `apply_block_missing_target_creates_file` (pre-image sha = sha(""))
- `apply_block_atomic_no_halfwrite` (proposed fully present)
- `revert_block_restores_pre_image`
- `revert_block_refuses_on_sha_mismatch` (corrupt backup → err, target unchanged)

### B4 — `stage_artifact` command + resolver
**Files:** `src-tauri/src/commands.rs`, `src-tauri/src/forge.rs` (small helper)
- `async fn stage_artifact(state, finding_id: Option<String>, issue_id: Option<String>) -> Result<Artifact, String>`: resolve a `Finding` from `finding_id` (store) or `issue_id` (orb scene, mirror `get_orb_fix_preview`); build the fix-preview (`forge::preview_for_finding`) to capture `target_path`+`diff`; resolve the `block` via the same per-pattern fix; construct PENDING `Artifact`; `store.save_artifact`; return it. Re-stage of an existing non-applied artifact for the same `(finding_id, target_path)` returns the existing row.
**Acceptance:** returns a PENDING artifact with the resolved absolute target + non-empty (or empty-if-present) diff + stored block.
**Tests:** stage from a saved finding (temp db) → row persisted, status `pending`, `target_path` honors `WARDEN_CLAUDE_MD` temp override.

### B5 — Implement `apply_artifact` / `revert_artifact` (replace stubs)
**Files:** `src-tauri/src/commands.rs`
- `apply_artifact(state, id: String) -> Result<Artifact, String>`: load row (404 if absent); `forge::apply_block` against `artifact.block` into `backup_dir/<id>.bak`; `store.update_artifact_status(id, "applied", now, backup_path, pre_image_sha)`; return updated artifact. No-op (changed=false) still records `applied` with null backup.
- `revert_artifact(state, id: String) -> Result<Artifact, String>`: load row; if not applied → Ok no-op (return row); else `forge::revert_block(target, backup_path, pre_image_sha256)`; `update_artifact_status(id, "reverted", …)`; return.
- Keep the `(id: String)` arg signature; widen return `()` → `Artifact`.
**Acceptance:** apply writes the temp target once; revert restores it; both return the updated artifact with correct status.
**Tests:** command-level happy path against temp db + `WARDEN_CLAUDE_MD` temp target: stage→apply (file has one block)→revert (file restored). Double-apply is a no-op.

## Acceptance Summary
- Apply is a backup+write of `ensure_block` output — no patch engine, idempotent, re-entrant.
- Backups stored beside target, namespaced by artifact id; pre-image sha recorded.
- Revert verifies sha before restoring; refuses on mismatch.
- Artifact lifecycle pending→applied→reverted persisted and survives restart.
- Every detector pattern keeps its own meaningful guardrail (wildcard is a net, not the strategy).
- FACE delivers the cinematic approve/apply/revert loop with honest, status-driven state.
- No test or smoke ever writes the real `~/.claude/CLAUDE.md`.
