# WARDEN M2 — "Face" Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` (recommended)
> or `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax.
> Frontend tasks MUST invoke `r3f-mastery`, `remotion`, and `frontend-design` skills. Verify every task with
> the stated build/test commands before checking it off — `superpowers:verification-before-completion`.

**Goal:** Turn WARDEN into an always-on macOS menubar daemon summoned by ⌘⇧Space that drops a pre-warmed
green-on-black overlay, runs the existing Fugu pipeline on real Claude + Codex transcripts, drives an R3F +
Remotion "war-room" from honest pipeline signals, and renders a ranked, evidence-cited diagnosis with
read-only fix previews — without breaking anything M0/M1 already ships.

**Architecture:** One Tauri v2 process as an `Accessory` (menubar) app. Rust core: an `Adapter` registry
(Claude + new Codex) with backfill + live FSEvents byte-offset tailing → rusqlite/FTS5 store → env-swappable
Fugu `Brain` emitting honest viz events → IPC. Frontend: vanilla-TS overlay shell (boot, terminal, HUD,
router, diagnosis/evidence/fix-preview DOM) + a React/R3F/Remotion island mounted once into `#war-room-root`
on the hidden pre-warmed window.

**Tech Stack:** Rust (tauri 2.9, rusqlite, tokio, `notify`, reqwest SSE), TypeScript + Vite, React 18,
@react-three/fiber + three + drei + postprocessing, @remotion/player, animejs.

## Global Constraints
- macOS Apple Silicon; Tauri v2.9.5; `macos-private-api` already on. Package manager: **pnpm**.
- **Preview-only milestone:** no writes to user projects; fix preview renders diffs only; apply = M4.
- **No `git push` / PR / MR / outbound publishing** without explicit instruction. Local commits are fine.
- **No CDN** — bundle every dep through Vite.
- **Honest viz:** every node/flare maps to a real signal. Off-Fugu engines (no `orchestration_*`) degrade to
  delta pulses + plain token weight; never fabricate per-agent theater.
- **Harness is first-class:** carried on every `Session`, surfaced in every UI element that shows a
  session/finding, via one `harness_theme`/`harnessTheme` source. Color always paired with glyph + label.
- **Defensive ingest:** unknown record → `Event::SystemNotice`; a failing adapter never stalls the others.
- **Engine path/shape stays Responses-API** (`/responses`); only base URL + models + key + effort are env-swept.
- Don't regress M0/M1: `cargo test` must stay green; the existing Claude mapping is unchanged.

pub trait Adapter: Send + Sync {
    fn harness(&self) -> Harness;

    /// `bytes` is the whole file and the batch carries full session context. When >0, `bytes` is the
    /// appended tail; events carry ABSOLUTE RawRef offsets (start_offset + local) and attach to the
    /// session derived from `path`. Default impl: full-parse only (offset 0). Codex+Claude override.

        -> Result<Vec<SessionBatch>>;

impl AdapterRegistry {
    pub fn new(store: Store) -> Self;            // registers ClaudeCodeAdapter + CodexAdapter (Task 3)
    pub fn adapters(&self) -> &[Box<dyn Adapter>];
    pub fn backfill_all(&self, store: &Store) -> IngestSummary; // isolates per-adapter failure (never panics)
}
pub struct IngestSummary { pub by_harness: Vec<(Harness, usize, usize)>, pub errors: Vec<String> }
